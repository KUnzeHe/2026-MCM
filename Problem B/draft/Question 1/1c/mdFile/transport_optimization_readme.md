
# 运输系统优化模型说明（混合方案：电梯链路 + 地面直达火箭）

## 0. 模型定位与核心目标
本模型用于“混合方案” ( Scenario 1c )，旨在探索 **时间 (Timeline)** 与 **成本 (Cost)** 之间的权衡关系。

模型满足以下核心特征：
1. **并行系统**：电梯链路与地面火箭并行工作，完工时间由较慢者决定。
2. **瓶颈与离散性**：电梯系统受瓶颈环节限制，火箭运输考虑离散批次取整。
3. **多目标优化视角**：通过设定不同的完工时间上限 $Y_{\max}$，求解对应的最小成本，从而绘制 **Pareto 前沿 (Time-Cost Curve)**。

混合方案含两条并行分支：
- 分支 A（电梯链路）：地面 → 电梯 → 锚点 → 转运火箭 → 月球
- 分支 B（地面直达火箭）：地面 → 传统火箭 → 月球

---

## 1. 符号与决策变量
- $M_{\text{tot}}$：总运输质量（吨）
- $x$：走电梯链路的质量（吨），$0\le x \le M_{\text{tot}}$
- $M_{\text{tot}}-x$：走地面直达火箭的质量（吨）

电梯链路参数：
- $T_E$：电梯年吞吐能力（吨/年）
- $T_{R,\text{anchor}}$：锚点转运火箭年吞吐能力（吨/年）
- $p_A$：锚点转运火箭单次有效载荷（吨/次）
- $T_{\text{chain}}=\min(T_E, T_{R,\text{anchor}})$：电梯链路瓶颈吞吐能力（吨/年）
- $F_E$：电梯链路固定成本
- $c_E$：电梯链路单位成本

地面直达火箭参数：
- $p_B$：地面直达火箭单次有效载荷（吨/次）
- $N_{\text{sites}}$：发射场数量
- $L_{\max}$：单场年最大发射次数
- $T_R=N_{\text{sites}}L_{\max}p_B$：地面直达火箭年吞吐能力（吨/年）
- $c_R=C_{\text{launch}}/p_B$：地面直达火箭单位成本（$/\text{吨}$）

### 1.1 参数赋值表（基准情景）

| 参数 | 含义 | 基准值 | 来源/假设 |
|------|------|--------|-----------|
| $M_{\text{tot}}$ | 总运输质量 | $10^8$ 吨 | 题目给定 |
| $T_E$ | 电梯年吞吐能力 | 537,000 吨/年 | 题目：179,000 吨/年 × 3 Harbours |
| $N_{\text{anchor}}$ | 锚点数量 | 6 | 题目：3 Harbours × 2 anchors/Harbour |
| $p_A$ | 锚点火箭单次载荷 | 500 吨 | 假设（太空发射效率高） |
| $L_{\text{anchor}}$ | 锚点年发射次数 | 200 次/年 | 假设 |
| $p_B$ | 地面火箭单次载荷 | 125 吨 | 题目：100–150 吨取中值 |
| $N_{\text{sites}}$ | 地面发射场数量 | 10 | 题目给定 |
| $L_{\max}$ | 单场年最大发射次数 | 50 次/年 | 行业参考（SpaceX 2024 ~100次/年） |
| $C_{\text{launch}}$ | 单次发射成本 | \$150M | Falcon Heavy 约 \$97M，保守估计 |
| $c_R$ | 地面火箭单位成本 | \$1,200/kg | $= C_{\text{launch}} / p_B$ |
| $c_E$ | 电梯链路单位成本 | \$50/kg | 假设（主要为电力+锚点转运燃料） |
| $F_E$ | 电梯系统固定成本 | \$100B | 假设（已建成，可视为沉没成本） |

> **注**：上述参数为初始基准值，后续敏感性分析将对 $T_E$、$c_R$、$L_{\max}$ 等关键参数进行扰动。

## 2. 批次完成时间（离散发射）
电梯链路：
$$Y_A(x)=\max\left(\frac{x}{T_E},\frac{\lceil x/p_A\rceil}{N_{\text{anchor}}L_{\text{anchor}}}\right)$$
其中 $N_{\text{anchor}}L_{\text{anchor}}$ 表示锚点转运火箭可用发射次数/年（若已内化在 $T_{R,\text{anchor}}$，可用等价形式 $\lceil x/p_A\rceil/T_{R,\text{anchor}}^{(\text{launch})}$）。

地面直达火箭：
$$Y_B(x)=\frac{\lceil(M_{\text{tot}}-x)/p_B\rceil}{N_{\text{sites}}L_{\max}}$$

总体完工时间：
$$Y(x)=\max\left(Y_A(x),Y_B(x)\right)$$

---

## 3. 关键约束：限定时间 $Y_{\max}$
要求：
$$Y(x)\le Y_{\max}$$

用保守边界表达可行区间：
$$\lceil x/p_A\rceil\le N_{\text{anchor}}L_{\text{anchor}}\,Y_{\max}$$
$$\lceil(M_{\text{tot}}-x)/p_B\rceil\le N_{\text{sites}}L_{\max}\,Y_{\max}$$

若用连续近似，可写为：
$$x\le T_{\text{chain}}Y_{\max}$$
$$M_{\text{tot}}-x\le T_RY_{\max}$$

可行性判别（连续近似）：
$$Y_{\max}\ge \frac{M_{\text{tot}}}{T_{\text{chain}}+T_R}$$

---

## 4. 成本模型（限时条件下）
引入 $0-1$ 变量 $y$ 表示是否启用电梯链路：
$$y = \begin{cases} 1, & \text{if } x > 0 \\ 0, & \text{if } x = 0 \end{cases}$$

总成本函数：
$$C(x,y)=yF_E+c_E x+c_R(M_{\text{tot}}-x)$$
逻辑约束：
$$0\le x\le y \cdot M_{\text{tot}}$$
同时需满足第 3 节的时间约束。

**求解策略**：
在线性成本下（假设 $c_E < c_R$），给定 $Y_{\max}$，为了最小化成本，应**极大化电梯链路的运量 $x$**。
即取 $x$ 在时间约束允许下的最大值：
$$x^* = \min(M_{\text{tot}}, \lfloor T_{\text{chain}} Y_{\max} \rfloor_{\text{batch}})$$
（需注意离散批次取整的影响）

---

## 5. 混合方案求解流程
1. **确定时间范围**：计算理论最短时间 $Y_{\min}$ (全系统拉满) 和最长时间 $Y_{\text{pure\_elevator}}$ (纯电梯)。
2. **遍历 $Y_{\max}$**：在 $[Y_{\min}, Y_{\text{pure\_elevator}}]$ 区间内离散化选取多个 $Y_{\max}$ 样本点。
3. **求解最优 $x$**：对每个 $Y_{\max}$，计算满足约束的最大 $x$（使得低成本的电梯分担更多）。
4. **输出结果**：获得一组 $(Y_{\max}, C_{\min})$ 数据对，绘制曲线。

---

## 6. 进阶分析：时间-成本权衡 (Pareto Frontier)
由于题目未给出硬性截止时间，本模型的核心产出应是一条 **Pareto 前沿曲线**：
*   **左端点 ($Y_{\min}$)**：成本极高。依靠大量昂贵的地面火箭来缩短工期。
*   **右端点 ($Y_{\text{pure\_elevator}}$)**：成本最低。虽慢，但利用了低边际成本 $c_E$ 且摊薄了固定成本 $F_E$。
*   **拐点分析**：曲线可能存在“膝点 (Knee Point)”，即稍微增加一点时间就能大幅降低成本的黄金平衡点。

---

## 7. 模型扩展性
1.  **环境影响扩展**：
    成本 $C$ 可定义为广义成本，包含环境税或碳排放折价：
    $$C_{\text{total}} = C_{\text{financial}} + \lambda \cdot E_{\text{emission}}$$
    其中 $\lambda$ 为碳价。这将使得低污染的电梯方案更具优势。
2.  **敏感性分析接口**：
    *   **$T_E$ (电梯运力)**：若 179,000 吨/年 仅为单港口数据，则总运力翻倍，时间显著缩短。
    *   **$c_R$ (火箭发射成本)**：若未来火箭复用技术成熟，$c_R$ 下降，Pareto 曲线将整体下移，且最优策略可能向火箭偏移。

---

## 8. 核心结论摘要
1. **决策本质是权衡**：不存在单一最优解，而是“花钱买时间”的权衡博弈。
2. **电梯的优势在于长周期**：只有当规划周期够长，足以让 $x$ 累积得很大时，巨大的固定成本 $F_E$ 才值得投入。
3. **混合方案的价值**：在电梯运力不足（瓶颈明显）时，利用火箭作为补充，可以在不大幅增加总成本的前提下，有效“切削”掉完工时间的尾部（比如最后 5% 的物资若等电梯需很久，不如直接发火箭）。
4. **若不可行**，应优先增加发射次数、提升 $p$ 或扩建发射场/电梯能力。
